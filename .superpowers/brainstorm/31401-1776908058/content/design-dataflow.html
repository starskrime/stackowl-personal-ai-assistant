Design: Data Flow

Section 2 of 3 — How messages move through the system

Inbound — User types a message
InputHandler captures raw stdin keystrokes │ buffers chars, handles cursor/history/backspace │ on Enter → emits "line" event with text string ▼ CLIAdapter receives "line" event │ wraps into GatewayMessage { id, channelId:"cli", userId, sessionId, text } │ pushes to serial queue (one message at a time) │ calls ui.showThinking() on renderer ▼ OwlGateway .handle(msg, callbacks) │ callbacks.onStreamEvent → renderer.onStreamChunk(chunk) │ callbacks.askInstall → renderer.promptYesNo() → InputHandler ▼ GatewayResponse { content, owlName, owlEmoji, toolsUsed, usage }
Outbound — Gateway responds
GatewayResponse ▼ CLIAdapter .sendToUser() / .broadcast() │ calls renderer.showResponse(response) ▼ TerminalRenderer │ updates RightPanel state (pushes message to conversation) │ updates TopBar state (turn count, tokens, cost) │ unlocks InputBox │ schedules redraw() ▼ Components each renders its ANSI string from current state ▼ process.stdout.write() ← only the renderer touches stdout
Streaming — Real-time token output
OwlGateway fires onStreamEvent for every text_delta ▼ TerminalRenderer.onStreamChunk(chunk) │ appends chunk to RightPanel conversation buffer │ schedules redraw() (debounced — no render storm) ▼ RightPanel re-renders with latest buffer content ▼ stdout user sees tokens appear in real time
Redraw Loop
Any state change → renderer.redraw() │ if already queued → skip (dedupe) │ setImmediate() → batch multiple state changes │ if already rendering → skip (no re-entrancy) ▼ renderer builds full frame string: frame() + topBar() + leftPanel() + rightPanel() + inputBox() + cmdPopup() + shortcutsBar() ▼ single process.stdout.write(fullFrame) ← one write per frame
Home → Session Transition
renderer starts in home mode LeftPanel renders: identity + backend info RightPanel renders: recent sessions + centered input User submits first message → renderer.setMode("session") LeftPanel switches to: mind state + DNA + tool trace RightPanel switches to: conversation history InputBox moves to bottom No HomeScreen class needed — renderer handles both modes
