StackOwl — Platform Flow Map

Complete architecture · 30+ subsystems · Status: code-present (not validated end-to-end)

Message Pipeline (happy path)

Inbound Channel receives message → GatewayMessage created → SessionManager load/create → RoutingCoordinator (owl selection) → ContextBuilder (memory + pellets + skills) → OwlEngine ReAct loop → Tool calls (0..N) → Final response text → PostProcessor (save memory, queue evolution) → Channel sends reply
ReAct Loop detail PromptOptimizer (build system prompt) → ModelRouter (pick provider + model) → LLM call (stream) → Tool intent router → select relevant tools → Execute tool → Observe result ↩ repeat max 15x
Post-process (async) FactExtractor → FactStore → EpisodicMemory update → MicroLearner signals → DNA evolution queue → PelletGenerator (if complex) → UserPreferenceModel update
