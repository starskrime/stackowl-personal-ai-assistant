## Critical Findings From Gaps Pass

### Finding G-01: Evolution Subsystem Is Almost Entirely Side-Only

`src/evolution/` has 16 files. Only `reflexion.ts` is wired to the gateway message path (post-response, called by PostProcessor). The remaining 15 files — including `desire-executor.ts`, `synthesizer.ts`, and 13 support files — run only via the 5-minute background tick in `BackgroundOrchestrator`. This means:

- Self-evolution happens on a timer, not tied to actual conversations
- The system synthesizes desires and executes them whether or not any conversation happened that day
- Reflexion (the one wired piece) ties learning to real interactions, but the broader evolution loop doesn't

**Recommendation:** Wire `desire-executor` to session-end events (after PostProcessor completes). Cap at 30s timeout per session. This connects evolution to actual interactions without changing the background tick.

### Finding G-02: The GatewayContext Injection Pattern Is Nearly Free

Week 2 discovered that `GatewayContext` already holds references to `socraticEngine`, `questManager`, `groundState`, `timelineManager`, and others. Week 3 gap scoring confirmed: for each of these gaps, the wiring cost is **only in the handler layer**, not in dependency injection. The instantiation is done; the object is in scope; it just needs to be called.

This changes the effort estimates for GAP-001, GAP-002, GAP-004, GAP-005 significantly downward. Most are 1–3 hours of wiring work, not days.

### Finding G-03: Clarification Subsystem Is 40% Orphaned

`src/clarification/` has 10 files. 7 are wired (intent clarifier, slot filler, proactive clarifier, clarification router, session debrief, loop detector, unclarity surfacer via indirect path). 3 are confirmed orphans:
- `ambiguity-detector.ts` — not called before strategy classification
- `pre-execution-confirmer.ts` — not called at tool execution time
- `unclarity-surfacer.ts` — unclarity-surfacer.ts listed as orphan but may be called via indirect path (needs grep verification)

### Finding G-04: Discord/WhatsApp/Voice Are Present But Unconfirmed Active

Files exist: `src/gateway/adapters/discord.ts`, `src/gateway/adapters/voice.ts`. No evidence of active bot tokens or connection code being bootstrapped in `src/index.ts`. These adapters may be feature-complete but require configuration to activate. WhatsApp adapter was not found at all — Epic 4 plan may have been partially aborted.

### Finding G-05: `forge/` and `tournaments/` Are Truly Aspirational

`src/forge/` (4 files): Never instantiated anywhere. The demo→skill synthesis concept exists only in code; no wiring path exists. `src/tournaments/` (3 files): Referenced only by type imports. Both are confirmed aspirational and should not be scheduled until the platform's core gaps close.
